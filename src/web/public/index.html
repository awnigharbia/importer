API Endpoints

POST /api/import

Start a new import job to download a video and upload it to Bunny Storage.

Request Body:

{
  "url": "https://drive.google.com/file/d/ABC123/view",
  "type": "gdrive" | "direct",  // Optional, auto-detected if not provided
  "fileName": "my-video.mp4"    // Optional
}

Response:

{
  "success": true,
  "data": {
    "jobId": "abc123",
    "requestId": "req_xyz789",
    "status": "waiting",
    "url": "https://drive.google.com/file/d/ABC123/view",
    "type": "gdrive"
  }
}

GET /api/jobs

List all jobs with pagination and filtering options.

Query Parameters:

  • page - Page number (default: 1)
  • limit - Items per page (default: 20)
  • status - Filter by status: completed, failed, active, waiting, delayed

Response:

{
  "success": true,
  "data": {
    "jobs": [...],
    "pagination": {
      "page": 1,
      "limit": 20,
      "total": 100,
      "totalPages": 5
    },
    "counts": {
      "completed": 80,
      "failed": 5,
      "active": 2,
      "waiting": 13,
      "delayed": 0
    }
  }
}

GET /api/jobs/:id

Get detailed information about a specific job.

Response:

{
  "success": true,
  "data": {
    "id": "abc123",
    "data": {...},
    "status": "completed",
    "progress": {
      "stage": "uploading",
      "percentage": 100,
      "message": "Upload complete"
    },
    "returnValue": {
      "success": true,
      "cdnUrl": "https://cdn.example.com/video.mp4",
      "fileName": "video.mp4",
      "fileSize": 104857600
    },
    "attemptsMade": 1,
    "timestamp": 1234567890000
  }
}

POST /api/jobs/:id/retry

Manually retry a failed job.

Response:

{
  "success": true,
  "message": "Job retry requested successfully"
}

DELETE /api/jobs/:id

Delete a job from the queue.

Response:

{
  "success": true,
  "message": "Job deleted successfully"
}

TUS Upload

The service supports resumable uploads using the TUS protocol at /uploads.

Use any TUS client to upload files directly to the service.

Environment Variables

Configure the service using the following environment variables:

# Bunny Storage
BUNNY_STORAGE_ZONE=your-zone
BUNNY_ACCESS_KEY=your-access-key
BUNNY_CDN_URL=https://your-cdn.b-cdn.net

# Redis
REDIS_URL=redis://localhost:6379

# Sentry (optional)
SENTRY_DSN=your-sentry-dsn

# Telegram (optional)
TELEGRAM_BOT_TOKEN=your-bot-token
TELEGRAM_CHAT_ID=your-chat-id

# Server
PORT=3000
TUS_PATH=/uploads
TEMP_DIR=./temp
